EAS project secrets (run from repo root).

Install / run the CLI (Windows has no global `eas` until you add one):
  npm install
  npx eas login
  npx eas update --branch production --message "Your message"
  # or: npm run update:production -- --message "Your message"

Use the same names the app reads at build time (EXPO_PUBLIC_* are inlined into the JS bundle).

  npx eas secret:create --scope project --name EXPO_PUBLIC_SUPABASE_URL --value "https://YOUR_PROJECT.supabase.co"
  npx eas secret:create --scope project --name EXPO_PUBLIC_SUPABASE_ANON_KEY --value "YOUR_ANON_JWT"
  npx eas secret:create --scope project --name EXPO_PUBLIC_REVENUECAT_IOS_KEY --value "YOUR_RC_PUBLIC_IOS_KEY"
  npx eas secret:create --scope project --name EXPO_PUBLIC_REVENUECAT_ANDROID_KEY --value "YOUR_RC_PUBLIC_ANDROID_KEY"
  npx eas secret:create --scope project --name EXPO_PUBLIC_SENTRY_DSN --value "https://YOUR_KEY@oORG.ingest.sentry.io/PROJECT"

Optional (Sentry source maps / debug symbols on EAS — not the same as DSN):

  npx eas secret:create --scope project --name SENTRY_AUTH_TOKEN --value "YOUR_SENTRY_AUTH_TOKEN"

After first link:

  npx eas init

This adds extra.eas.projectId to app.json so OTA updates.url is set via app.config.ts.

Reference: https://docs.expo.dev/eas/environment-variables/
