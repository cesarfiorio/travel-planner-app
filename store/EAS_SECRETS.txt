EAS project secrets (run from repo root, logged in: npx eas login).

Use the same names the app reads at build time (EXPO_PUBLIC_* are inlined into the JS bundle).

  eas secret:create --scope project --name EXPO_PUBLIC_SUPABASE_URL --value "https://YOUR_PROJECT.supabase.co"
  eas secret:create --scope project --name EXPO_PUBLIC_SUPABASE_ANON_KEY --value "YOUR_ANON_JWT"
  eas secret:create --scope project --name EXPO_PUBLIC_REVENUECAT_IOS_KEY --value "YOUR_RC_PUBLIC_IOS_KEY"
  eas secret:create --scope project --name EXPO_PUBLIC_REVENUECAT_ANDROID_KEY --value "YOUR_RC_PUBLIC_ANDROID_KEY"
  eas secret:create --scope project --name EXPO_PUBLIC_SENTRY_DSN --value "https://YOUR_KEY@oORG.ingest.sentry.io/PROJECT"

Optional (Sentry source maps / debug symbols on EAS — not the same as DSN):

  eas secret:create --scope project --name SENTRY_AUTH_TOKEN --value "YOUR_SENTRY_AUTH_TOKEN"

After first link:

  npx eas init

This adds extra.eas.projectId to app.json so OTA updates.url is set via app.config.ts.

Reference: https://docs.expo.dev/eas/environment-variables/
